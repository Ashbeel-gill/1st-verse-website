## Animation architecture (`app.js`)

**Hard performance contract** (from the original brief â€” do not break):
- GSAP animates **only** transforms (`x/y/z/scale/rotation/rotationX/rotationY`) and `opacity`/`autoAlpha`. Never width/height/top/left. The chart reveal uses a scaleX curtain, not path/dashoffset. Card hover shadows animate via pre-rendered layers/box-shadow CSS transitions, not per-frame shadow tweens.
- `gsap.matchMedia()` with three contexts: desktop (â‰¥768px) gets tilt/parallax/aurora/magnetic/glare/idle-floats; mobile gets lightweight opacity fades and static layout (chips become static elements, some hidden); `prefers-reduced-motion` gets a fully static page (CSS block at the bottom of `styles.css` + early return in JS).
- Event listeners registered inside contexts are cleaned up via `context.add(() => () => â€¦)`.
- Idle floats use `yPercent` so they never fight parallax/tilt `x/y` (GSAP keeps the channels separate).
- Elements pre-hidden for choreography via `.js [data-reveal] / [data-stagger] / [data-convo-item] / [data-show-*] / [data-v-pop] / [data-feat-pop] / [data-cmp-pop]` CSS (the `js` class is added by app.js; no-JS users see everything).

Key systems, in file order: `countUp` utility â†’ hero intro timeline â†’ conversation loop â†’ scroll reveals (singles) â†’ `ScrollTrigger.batch` for grids â†’ showcase scene timelines â†’ **how-it-works theatre** (`initTheatre` â€” pinned, scrubbed master timeline + cumulative pipeline; the pin has `refreshPriority: 1` so it's measured before the triggers below it) â†’ **feature micro-demos** (`initFeatureDemos` â€” one scroll-triggered demo per bento card) â†’ **comparison table** (`initCompare` â€” row-by-row tick cascade) â†’ aurora drift â†’ hero scene 3D tilt â†’ card 3D tilt + glare â†’ idle floats â†’ magnetic CTAs â†’ mobile nav â†’ FAQ expansion â†’ **Cal.com fallback** (if embed.js fails to load, `[data-cal-link]` clicks open `cal.com/â€¦` directly â€” detection via script error event + a 4s queue-drain check) â†’ calculator â†’ chat widget.

## Dev & verification

- Preview: `preview_start` with the `1st-verse` config (`.claude/launch.json`, `npx serve` on port 4173), or any static server.
- **Preview-panel quirks** (harness, not the site): the tab is throttled â€” GSAP lag-smoothing makes animations crawl and CSS transitions freeze, `requestAnimationFrame` starves entirely, and `preview_screenshot` intermittently hangs until the server is stopped/restarted. Judge motion only in a focused real browser tab. Prefer `preview_eval` + computed-style probes for verification.
- Emulated `preview_resize` does **not** fire matchMedia change events â€” reload after resizing to re-run GSAP contexts.
- The Cal.com embed is blocked in the sandboxed preview (`ERR_BLOCKED_BY_RESPONSE`, cross-origin isolation); it works in a normal browser. The JS fallback covers real-world failures.
- Breakpoints to check: 1280 (full), ~820 (tablet, burger appears â‰¤860), 375 (mobile). Always check `document.documentElement.scrollWidth > clientWidth` for overflow regressions â€” absolutely-positioned floaters are the usual culprit.
- Known footgun: scoping selectors like `.stat span` or `.ledger__row span` too loosely catches nested number spans â€” use `>` child combinators around the ledger/stats.
- **Token economy** (the transcript is the main drain â€” it re-processes every turn): edit via `Grep` â†’ targeted `Read(offset/limit)` â†’ `Edit`, never full-file re-reads; batch verification into a single `preview_eval` rather than many round-trips; and `/clear` (or a fresh session) between distinct work chunks â€” code + this file persist, so nothing is lost.
- Verifying **scroll-scrubbed motion** (theatre, feature demos) in the throttled preview: the GSAP ticker is starved, so nothing plays on its own. Drive it directly â€” `ScrollTrigger.getAll().filter(t=>t.pin)[0].animation.progress(p)` for the theatre â€” or force-complete tweens (`gsap.getTweensOf(el).forEach(t=>t.progress(1))`) and read computed styles. `preview_resize` reflows CSS grid/media queries immediately (bento breakpoints are pure CSS), but JS `matchMedia` contexts still need a reload.
